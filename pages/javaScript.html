    WebExcel

    About

    SPA analogue of Google Sheets. The layout is made by myself. The application is written in pure JavaScript. During the implementation of the project, analogues of Redux, small library for manipulating the DOM and a small framework for Excel components were written. At the moment there is only a desktop version and not all the planned functionality has been implemented, but the main part of the application has been implemented. The code is written with an object-oriented approach, procedural and functional approaches are also used.

    Functionality

    • creating and deleting user tables
    • table name change
    • resizing rows and columns
    • changing cell styles
    • changing the styles of the selected parts of the text inside the cells (this functionality is not finished yet and may not always work correctly)
    • selection of a group of cells
    • copying the values of the selected cell or group of cells by dragging the slider in the lower right corner of the selection
    • creating calculated values in cells. To do this, use a line like '=(A10 * E2) + C5' (recalculation of the values of cells dependent on each other after changing the value of any of them has not yet been implemented)
    • keyboard control similar to Google Sheets

    Details

    • created a core for writing an application
      • Emitter class for handling custom events
      • DomListener class is responsible for handling native component events
      • ExcelComponent class inherits from DomListener and is responsible for initializing and deleting the component, as well as for working with Emitter's events and working with the Redux store analogue. This is the base class for Excel components
      • ExcelStateComponent class inherits from ExcelComponent and adds internal state to the component. This is the base class for stateful Excel components
      • Page class is the interface of the application's page classes
      • Router class is responsible for page routing uses History API
      • a library was written to simplify working with the DOM
    • an analogue of Redux was written
    • Storage class was written to work with LocalStorage. User-created tables are stored in LocalStorage
    • written Excel component class responsible for initializing and destroying all Excel components, creating a common store and emitter
    • Excel component classes are written. Header, Toolbar, Formula, Table classes and their inner classes. For example inner classes for Table class are Resizer, Selection, MathParser and other classes
    • ExcelPage and DashbordPage classes for application pages are written
    • JEST framework was used to write unit tests
    • code style is maintained by Eslint with Airbnb Style Guide and Prettier
    • Webpack is used to build the project (Babel, image compression, SCSS preprocessing)

    Funiro

    About

    A one-page online store with a shopping cart, favorites and authorization. The layout is made by myself and adapted to any screen size. Ready-made solutions such as Firebase, Swiper.js and lightGallery were used. The code uses procedural and object-oriented approaches

    Details

    • user authorization with Firebase Authentication
    • dynamic loading of products in the products section, cart and favorites from Firestore Database and images for products from Firebase Storage
    • only authorized users have access to the cart and the state of the user's cart is stored in the Firestore Database
    • all users have access to favorites and the state of favorites is saved in LocaleStorage
    • Window class was created for the authorization window, mobile search and menu burger that allows you to hide and show windows, as well as hide already open windows when opening a new one
    • the WindowList class was created for shopping cart and favorites. It inherits from Window and allows you to monitor the state of the list using MutationObserver. If the list is empty, show the corresponding text
    • three sliders implemented with Swiper.js
    • implemented animation of image gallery movement depending on the position of the mouse cursor. Image browsing implemented with lightGallery
    • implemented submenu display on hover for non-touch screen devices and on tap for touch screen devices, as well as a burger menu for devices with small screens
    • drop-down menus implemented in the burger menu and footer
    • Webpack is used to build the project (Babel, image compression, SCSS preprocessing)
